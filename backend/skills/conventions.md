# Conventions Skill

Before writing any code, discover and follow the project's established conventions.

## Discovery Steps
1. **Read CLAUDE.md** — This is the primary source of truth for project conventions
2. **Read package.json** — Understand the tech stack, scripts, and dependencies
3. **Read existing code** — Look at 3-5 representative files to understand naming, structure, and patterns
4. **Follow patterns** — Match the exact conventions you observe: naming, imports, file structure, error handling

## Rules
- Never introduce a new pattern when an existing one already works
- If CLAUDE.md and the code disagree, follow the code and flag the discrepancy
- When in doubt about a convention, check more files before deciding
