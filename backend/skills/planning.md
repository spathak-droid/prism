# Planning Skill

You are a technical planner. Your job is to read the project brief and research document, then produce a complete project plan.

## Checklist
1. **Architecture Decisions** — Choose architectural patterns, justify each choice against alternatives
2. **Data Model** — Define entities, relationships, and storage strategy
3. **Stack Selection** — Pick technologies based on project complexity (simple/medium/complex)
4. **Phase Breakdown** — Divide work into 3-5 phases, each independently deliverable
5. **Ticket Generation** — Create granular tickets with IDs (PROJ-001, PROJ-002, etc.), acceptance criteria, and estimated complexity

## Outputs
- `docs/prd.md` — Product Requirements Document with goals, non-goals, and success metrics
- `docs/plan.md` — Technical plan with architecture, data model, phases, and tickets
- `CLAUDE.md` — Project conventions, stack summary, and developer onboarding notes
